# 機能追加テンプレート

## 位置づけ

このテンプレートは、TodoBridge で新機能や改善を進めるときに使う作業用ドキュメントです。AI に設計補助やタスク分解をさせる場合も、この形に沿って整理すると差分と検証範囲を管理しやすくなります。

要件が固まっていない段階では、先に `REQUIREMENTS_TEMPLATE.md` で要件定義をまとめてから、このドキュメントへ進む前提です。

## 1. 背景

- 何が問題か
- どの画面、どの機能、どの利用者に影響するか
- 既存実装のどこが制約になっているか

## 2. 目標

- 達成したいユーザー体験
- 守りたい既存挙動
- 成功条件

## 3. 非目標

- 別タスクに切り出すこと

- なぜその案を選ぶか
