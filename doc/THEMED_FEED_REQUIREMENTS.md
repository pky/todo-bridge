# テーマ選択型情報機能 要件定義

## 1. 背景

TodoBridge では、タスク管理に加えて関心テーマごとの情報収集を扱いたい。現在は AI / iOS / Android のような更新頻度の高いテーマを中心に扱っているが、将来的には育児、金融、教育、生活情報のようなテーマも追加したい。

ただし、すべてのテーマを同じ頻度、同じ見せ方で扱うと、情報量が多くなりすぎてノイズが増える。利用者や家族に必要なテーマだけを選び、テーマの性質に応じた頻度で届ける必要がある。

## 2. 解決したい課題

- 興味のないテーマまで同じ画面に並ぶと、必要な情報が埋もれる
- テーマ数が増えると固定タブ UI だけでは扱いづらい
- AI / iOS / Android のような高頻度テーマと、育児や金融のような週次で十分なテーマを同じ更新頻度で扱うのは非効率
- 集めた情報を読むだけで終わらせず、Todo に繋げたい
- 家族で共有したいテーマと、自分だけが追いたいテーマを分けたい

## 3. 目標

- 初回起動時に関心テーマを選択できる
- 選択したテーマだけをホームや情報画面で表示できる
- テーマごとに更新頻度を設定できる
- 高頻度テーマと低頻度テーマを同じモデルで扱える
- 情報から「あとで読む」「対応する」Todo へ接続できる
- 家族共有向きのテーマを扱える構造を持つ
- テーマ数が増えても AI 利用量と Firebase 利用量が増えすぎない

## 4. 非目標

- 初期段階ですべてのテーマを実装すること
- すべてのテーマをリアルタイム配信すること
- 外部ニュースサービスの完全な代替を目指すこと
- テーマごとに全く別の UI を持つこと

## 5. 想定ユーザー

- 主利用者: 開発者本人
- 拡張利用者: 家族と情報やタスクを共有したい小規模ユーザー

## 6. 想定利用シーン

- 初回起動時に、AI / iOS / Android のような高頻度テーマを選ぶ
- 将来的に、育児や金融のような週次テーマを追加する
- 家族で共有したいテーマの記事を見つけて、Todo に変換する
- 自分だけが追いたいテーマは個人設定として保持する
- 更新頻度の高いテーマは日次で、低いテーマは週次でまとめて確認する

## 7. 機能要件

### 7.1 テーマ選択

- 初回起動時に候補テーマ一覧を提示できる
- 利用者は複数テーマを選択できる
- 後から設定画面でテーマの追加、削除、並び替えができる

### 7.2 テーマ定義

- 各テーマは以下の属性を持つ
  - `id`
  - `表示名`
  - `カテゴリ`
  - `更新頻度`
  - `高頻度テーマかどうか`
  - `家族共有向きかどうか`
- AI / iOS / Android のような IT 系テーマと、それ以外のテーマを同じ仕組みで扱える

### 7.3 表示

- ホームでは選択済みテーマの情報を統合または優先表示できる
- 個別テーマの詳細表示ができる
- テーマ数が増えても固定タブだけに依存しない

### 7.4 更新頻度

- テーマごとに収集頻度と表示頻度を変えられる
- IT 系テーマは高頻度更新を前提とする
- 育児、金融などは週次中心で扱える

### 7.5 Todo 連携

- 情報項目を「あとで読む」Todo に変換できる
- 情報項目を「対応する」Todo に変換できる
- 家族に共有したい情報を Todo と同じ流れで扱える

## 8. 非機能要件

### 8.1 パフォーマンス

- 初回表示で全テーマを同時取得しない
- 高頻度テーマと低頻度テーマの更新を分離し、不要な更新を抑える

### 8.2 コスト

- テーマ数が増えても、AI API 呼び出し回数と Firebase 読み書き回数を抑える
- 無料枠で継続運用しやすいことを優先する
- 低頻度テーマは週次中心とし、毎日収集を前提にしない
- すべての記事に高コストな処理をかけず、必要なテーマや記事だけを優先する

### 8.3 保守性

- テーマ追加時に UI と収集基盤を全面改修しなくてよい構造にする
- 収集頻度や表示方針をテーマ定義から制御しやすくする

### 8.4 UX

- テーマ数が増えても画面遷移や一覧構造が破綻しない
- 必要な情報だけを見せ、ノイズを減らす

## 9. 制約

- 既存の AI / mobile ニュース機能を段階的に一般化する
- Todo 機能が主役であることは維持する
- 初期段階では AI / iOS / Android を優先し、他テーマは将来拡張とする
- AI と Firebase の使用量は無料枠で収まりやすい設計を優先する

## 10. 受け入れ条件

- 初回設定でテーマ選択ができる
- テーマ定義に更新頻度を持てる
- 選択済みテーマだけを表示できる
- 高頻度テーマと週次テーマを同じ仕組みで扱える
- 情報項目から Todo を作成できる

## 11. 未決事項

- ホーム画面を統合フィード中心にするか、テーマ別中心にするか
- 高頻度テーマの更新時刻や通知戦略をどうするか
